## Shared Patterns

These cross-cutting substitutions apply to ALL 9 migrated test files. Plans for individual test
files should cite this section rather than re-stating the table.

### Shared Pattern A — Import-line substitution (every test file)

**Source:** existing `test/*.test.mjs` headers (3 distinct header shapes seen).
**Apply to:** all 9 migrated files.

Replace the two `node:` import lines. The vitest imports must include exactly the hooks each file
already uses (do not add hooks a file does not use — lint `no-unused-vars` would flag them):

```javascript
// BEFORE — 3 header variants in the suite:
import { test } from 'node:test';                          // foundation, help (2 files)
import { test, afterEach } from 'node:test';               // ship, + verb files (4 files)
import { test, beforeEach, afterEach } from 'node:test';   // auth, config-profiles, auth-cli (3 files)
import assert from 'node:assert/strict';                   // all 9 files

// AFTER — one combined vitest import, mirror the hook set, add `expect`, drop assert:
import { test, expect } from 'vitest';                          // foundation, help
import { test, afterEach, expect } from 'vitest';               // ship, verb files
import { test, beforeEach, afterEach, expect } from 'vitest';   // auth, config-profiles, auth-cli
// (delete the `node:assert/strict` import entirely)
```

Everything below the import block (in-file helpers `captureLog`/`captureError`/`captureAll`/
`silentRun`, `beforeEach`/`afterEach` bodies, `mkdtemp` isolation, `installMockFetch`/`stubToken`
calls) is **unchanged** — they are runner-agnostic plain JS.

### Shared Pattern B — assert → expect substitution table (every test file)

**Source:** RESEARCH §Pattern 1; verified by grep against the 9 files (these six are the ONLY
methods used; counts confirmed live).

| `node:assert/strict` (count) | vitest `expect` replacement |
|------------------------------|------------------------------|
| `assert.equal(a, b)` (169) | `expect(a).toBe(b)` |
| `assert.match(s, re)` (65) | `expect(s).toMatch(re)` |
| `assert.deepEqual(a, b)` (27) | `expect(a).toEqual(b)` |
| `assert.doesNotMatch(s, re)` (17) | `expect(s).not.toMatch(re)` |
| `assert.ok(x)` (3) | `expect(x).toBeTruthy()` |
| `assert.rejects(fn, checker)` (2) | see Shared Pattern C (non-trivial) |

Concrete before/after from `test/foundation.test.mjs:55-58`:

```javascript
// BEFORE
assert.equal(obj.confirm_required, true);
assert.equal(obj.will, 'publish');
assert.deepEqual(obj.target_stores, ['apple_appstore']);
// AFTER
expect(obj.confirm_required).toBe(true);
expect(obj.will).toBe('publish');
expect(obj.target_stores).toEqual(['apple_appstore']);
```

`assert.ok` appears 3× — all in `ship.test.mjs:172` (`assert.ok(Array.isArray(out.steps))`) and
`config-profiles` (verify with grep). Use `.toBeTruthy()` to preserve exact semantics (NOT
`.toBe(true)` — `Array.isArray` returns boolean but other `ok` sites may be truthy-not-true).

### Shared Pattern C — `assert.rejects` port (the only non-mechanical case, 2 sites in `auth.test.mjs`)

**Source:** existing `test/auth.test.mjs:67-76` and `:121-132`.

The original `assert.rejects(fn, checker)` ran assertions INSIDE the checker. A naive
`.rejects.toThrow()` would DROP the `err.status === 401` / no-PAT-leak checks (false green —
RESEARCH Pitfall 4). Port to capture-then-assert so every inner assertion survives.

Site 1 — `auth.test.mjs:67-76` (checker inspects status, message match, and PAT-leak):
```javascript
// BEFORE
await assert.rejects(
  () => apiFetch('http://test.local', 'GET', '/api/v1/apps', null, 'production'),
  (err) => {
    assert.equal(err.status, 401);
    assert.match(err.message, /env 'production'.*appo login/);
    assert.doesNotMatch(err.message, /test-pat/);
    return true;
  },
);
// AFTER — capture the rejection, then assert each property (preserves all 3 checks)
let caught;
try {
  await apiFetch('http://test.local', 'GET', '/api/v1/apps', null, 'production');
} catch (e) {
  caught = e;
}
expect(caught).toBeDefined();
expect(caught.status).toBe(401);
expect(caught.message).toMatch(/env 'production'.*appo login/);
expect(caught.message).not.toMatch(/test-pat/);
```

Site 2 — `auth.test.mjs:126` (no checker, just "must reject", inside `captureAll`):
```javascript
// BEFORE
await assert.rejects(() => loginWithToken('http://test.local', 'staging', 'bad-pat'));
// AFTER
await expect(loginWithToken('http://test.local', 'staging', 'bad-pat')).rejects.toThrow();
```

### Shared Pattern D — per-file isolation replaces `--test-concurrency=1` (NO config setting)

**Source:** RESEARCH §Pattern 1 / Code Examples (Context7-confirmed vitest defaults).
**Apply to:** the whole suite; specifically delete the stale comment header in `auth.test.mjs:1-6`.

Vitest's default `isolate: true` + `pool: forks` gives each test FILE its own worker, `globalThis`,
and `process.env` snapshot — which is exactly what the old serial flag guaranteed. Therefore:
- **Drop** `--test-concurrency=1` (it lives only in `package.json` `test` script — replaced wholesale).
- **Do NOT** add `isolate`/`pool` to `vitest.config.mjs` — overriding `isolate: false` would
  RE-introduce the cross-file `globalThis.fetch` / `process.env.APPO_CONFIG_HOME` collision.
- **Delete** the now-false 6-line warning header at `auth.test.mjs:1-6` ("REQUIRES
  `--test-concurrency=1` ... serial ordering") — keep CLAUDE.md "no removed-code comments".
- **Keep** the per-test `beforeEach`/`afterEach` mkdtemp isolation EXACTLY as-is (it's app-level
  isolation via the lazy `configPath()`, not a runner feature).

## Pattern Assignments

### `vitest.config.mjs` (NEW config, test-runner)

**Analog:** No SDK file (the SDK ships no `vitest.config.*` — its `test`/`test:integration`/
`test:watch` scripts in `../sdk/package.json:52-54` work off vitest defaults). Use RESEARCH
§Pattern 2 minimal form. This is the one config with no direct file analog — flagged in "No
Analog" below.

**Recommended content (minimal):**
```javascript
// vitest.config.mjs
import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    environment: 'node',                 // CLI is Node-only; no jsdom
    include: ['test/**/*.test.mjs'],     // matches both test/unit/ and test/integration/
    globals: true,                       // optional — explicit imports kept, see below
    // isolate: true is the DEFAULT — intentionally NOT set (Shared Pattern D)
  },
});
```
**Note:** Since Shared Pattern A keeps explicit `import { test, expect } from 'vitest'` in every
file (lint-clean), `globals: true` is optional. Keep it only if the planner wants `test:integration`
glob runs to tolerate bare globals. The `include` glob is what makes `vitest run` find both subdirs
and `vitest run test/integration/` (D-05) target only integration.

### `tsconfig.json` (NEW config, static-analysis / typecheck)

**Analog:** `../sdk/tsconfig.json` (lines 1-18, read above).

**SDK source compilerOptions (the delta is large — SDK emits a built TS lib, we typecheck raw JS):**
```json
{ "target":"ES2020", "module":"ESNext", "moduleResolution":"bundler",
  "declaration":true, "declarationMap":true, "sourceMap":true,
  "strict":true, "esModuleInterop":true, "skipLibCheck":true,
  "forceConsistentCasingInFileNames":true, "outDir":"./dist", "rootDir":"./src" }
```

**Adaptation deltas (SDK bundled-TS → our raw-`.mjs` checkJs):**
| Concern | SDK value | CLI value | Reason |
|---------|-----------|-----------|--------|
| `allowJs` | (absent) | **`true`** | We have `.mjs` source, not `.ts` |
| `checkJs` | (absent) | **`true`** | Typecheck the JS via inference + JSDoc (D-08) |
| `noEmit` | (absent — SDK emits) | **`true`** | `tsc --noEmit`; no build (D-10) |
| `strict` | `true` | **`false`** | `src/*.mjs` has ZERO JSDoc tags today (verified) — strict floods implicit-any (Pitfall 1) |
| `module`/`moduleResolution` | `ESNext`/`bundler` | **`NodeNext`/`NodeNext`** | Raw `.mjs` under Node ESM, no bundler |
| `target` | `ES2020` | **`ES2022`** | Matches Node ≥18 (`engines.node >=18`) |
| `declaration`/`declarationMap`/`sourceMap`/`outDir`/`rootDir` | set | **DROP all** | No emit, no published types (CLI ships source) |
| `skipLibCheck`/`forceConsistentCasingInFileNames` | `true` | **KEEP** | Same hygiene |
| `types` | (absent) | **`["node"]`** | Needs `@types/node` so `node:fs`/`process`/`globalThis.fetch` type (A2) |
| `include` | `["src/**/*"]` | **`["bin/**/*.mjs","src/**/*.mjs","test/**/*.mjs"]`** | Lint+check the full JS surface (D-06 target list) |

**Recommended result (RESEARCH §Pattern 4):**
```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "allowJs": true,
    "checkJs": true,
    "noEmit": true,
    "strict": false,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "types": ["node"]
  },
  "include": ["bin/**/*.mjs", "src/**/*.mjs", "test/**/*.mjs"]
}
```
**Required devDep:** `@types/node` (RESEARCH A2) — without it `tsc --checkJs` errors on every `node:`
builtin. Pin a Node-18+-compatible major (`^20` or `^22`; latest is 25.x, but pin to match the floor).
**Wave-0 verification task:** after install, run `npx tsc --noEmit` to capture the REAL residual
error list (est. 0-15, RESEARCH Open Q1) and add JSDoc only for those — do not pre-write blanket
JSDoc tasks. The only `src/` changes this phase are these targeted JSDoc tags.

### `.github/workflows/ci.yml` (NEW config, CI)

**Analog:** `../sdk/.github/workflows/ci.yml` (lines 1-38, read above).

**SDK source shape:** `on: push/pull_request [main, master]` → single `build` job on `ubuntu-latest`
→ checkout@v4 → `pnpm/action-setup@v4 (v9)` → `setup-node@v4 (node 20, cache pnpm)` →
`pnpm install --frozen-lockfile` → `pnpm lint` → `pnpm typecheck` → `pnpm test` → `pnpm build`.

**Adaptation deltas (D-10, forced divergences):**
| SDK | CLI | Reason |
|-----|-----|--------|
| `pnpm/action-setup@v4` + `cache: 'pnpm'` + `pnpm install --frozen-lockfile` | **drop pnpm; `cache: 'npm'` + `npm ci`** | CLI is npm-based (`npm test`, `npm i -g @appolabs/appo`); `npm ci` needs the committed `package-lock.json` (Pitfall 2) |
| single `node-version: '20'` | **matrix `[18, 20, 22]`** | `engines.node >=18` — test the floor + current (D-10) |
| `pnpm build` step | **DROP** | CLI ships raw `.mjs`; no bundler (D-10) |
| `pnpm lint`/`typecheck`/`test` | **`npm run lint` / `npm run typecheck` / `npm test`** | same order, npm runner |
| triggers, checkout@v4, setup-node@v4, lint→typecheck→test order | **KEEP identical** | SDK parity (the north star) |

**Recommended result (RESEARCH §GitHub Actions CI):**
```yaml
name: CI
on:
  push:
    branches: [main, master]
  pull_request:
    branches: [main, master]
jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [18, 20, 22]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: 'npm'
      - run: npm ci
      - run: npm run lint
      - run: npm run typecheck
      - run: npm test
```
