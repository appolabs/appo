  <action>
    Create `.github/workflows/ci.yml` with the SDK-shaped, npm-adapted workflow:

    ```yaml
    name: CI
    on:
      push:
        branches: [main, master]
      pull_request:
        branches: [main, master]
    jobs:
      test:
        runs-on: ubuntu-latest
        strategy:
          matrix:
            node-version: [18, 20, 22]
        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-node@v4
            with:
              node-version: ${{ matrix.node-version }}
              cache: 'npm'
          - run: npm ci
          - run: npm run lint
          - run: npm run typecheck
          - run: npm test
    ```

    Then validate the exact CI step sequence locally (the live GitHub run is manual-only — see threat note). This proves the workflow's commands are green before pushing:
    ```bash
    npm ci && npm run lint && npm run typecheck && npm test
    ```
    `npm ci` requires the committed `package-lock.json` (Plan 01); if it errors "can only install with an existing package-lock.json", Plan 01's lockfile was not committed — stop and fix that first (Pitfall 2).
  </action>
